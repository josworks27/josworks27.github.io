# 1. 수업소개
* JOIN: 관계형 데이트베이스에서 가장 중요한 기능이라고 할 수 있음

# 2. 테이블 쪼개기
* 중복을 제거하는게 중요하다.
* 중복되는 값을 별도의 테이블로 분리하고 원래 테이블엔 식별할 수 있는 키를 준다.(포린 키 - 프라이머리 키)
* 모든 표는 하나의 주제만 가져야 한다!!(가장 중요한 개념!)
* 분산된 테이블은 읽기가 안 좋다. (하지만 쓰기가 좋다)
* 그래서 등장한 것이 관계형 데이터베이스의 JOIN 이다.

# 3. JOIN 실습준비
* 실습용 SQL - https://github.com/egoing/sql-join ,   JOIN의 종류 - https://sql-joins.leopard.in.ua/

# 4. LEFT JOIN
**LEFT OUTER JOIN**
* 가장 인기있는 JOIN
* 양쪽에서 매치되는 정보 뿐만 아니라 왼쪽에만 있는 정보도 출력
* 왼쪽 테이블의 정보에 조건이 맞는 오른쪽 테이블의 정보를 붙이는 것이다.

# 5. INNER JOIN
* 그냥 JOIN이라하면 INNER JOIN을 말함
* 왼쪽과 오른쪽 테이블 모두에 존재하는 정보만 가지고 만든다. 그래서 NULL이 존재하지 않음

# 6. FULL OUTER JOIN (별로 안 중요)
* 왼쪽에만 있는 것, 양쪽에 있는 것, 오른쪽에만 있는 것 모두를 출력한다.
* LEFT JOIN하고 RIGHT JOIN도 하고 중복되는 행은 삭제
* 실제로 쓰이는 경우는 별로 없음. 가볍게 개념정도 알면 좋음

# 7. EXCLUSIVE JOIN (별로 안 중요)
