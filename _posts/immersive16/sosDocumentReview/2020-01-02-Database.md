# Database란?
데이터: '내가 알고 싶어하는 것'에 관한 모든 사실들
데이터베이스: 데이터 보관과 조작이 가능한 구조화 된 데이터 모음

## SQL 이란?
데이터베이스는 웹 개발을 위해 다양한 정보를 효율적이고 안정적으로 보관하기 위해 사용한다.
가장 많이 사용하게 될 mysql이 SQL database(관계형 데이터베이스)고, 이를 활용하기 위한 언어가 SQL이다. SQL은 DB에 구조화된 질문을 할 수 있는 컴퓨터 언어이다.

## SQL vs NoSQL
1. SQL database - Strict Schema, Relations
  * Strict Schema: 스키마 설계 시 예상한 데이터보다 더 많은 데이터를 다루게 될 때 유연하게 스키마를 바꿀 수 없음. 이를 보완하기 위해 ORM을 사용할 수 있음(Sequelize CLI 등)
  * Relations: 스키마 설계 시 One to One, One to Many, Many to Many를 명확히 규정하고 스키마를 짜야 함. 중복되거나 잘못 입력된 데이터가 데이터베이스에 삽입되어서는 안된다.

2. NoSQL database(Not Only SQL) - No Schema, Relations
  * No Schema: JSON data와 같은 구조를 사용하여 자바스크립트의 객체를 다루는 것과 같은 느낌으로 데이터 처리가 가능함
  * No Relations: JOIN의 개념이 사라지고 duplicate된 데이터가 각각의 Collection에 저장됨(SQL의 table과 같은 개념). JOIN을 하지 않아도 각 Collection에서 원하는 정보를 조회할 수 있음

**Where to use SQL?**
* 초기 스키마 구성을 통해서 대부분의 데이터를 처리할 수 있을 때
* 스키마 구조는 비교적 간단하지만 Update가 많을 때
* Scale Up을 해야하는 경우가 적을 때 (서버의 성능이 향상되어야 할 때)

**Where to use NoSQL?**
* 이후에 어떤 데이터가 추가될 지 예상할 수 없을 때
* Read/Write가 Update 보다 현저히 많을 때
* Scale Up 해야하는 경우가 많을 때 (서버의 수량만 더 추가하면 OK)

# MVC 패턴이란?
* MVC 패턴은 애플리케이션을 Model, View, Controller 세 역할로 구분한 개발 방법론이다. 각 요소는 애플리케이션의 작동과 관련하여 각기 다른 역할을 수행한다.
  1. Model: 데이터 처리와 관련된 역할을 수행하는 파트이다. 컨트롤러의 명령을 받아 동작한다.
  2. View: 사용자들이 보는 화면(UI)으로 컨트롤러와 상호작용을 통해 데이터를 입력하거나 출력한다.
  3. Controller: 데이터(Model)와 사용자 인터페이스(View)를 연결하는 역할을 한다.

**구성요소를 분리하는 이유**
* 이해하기 쉽고 깔끔한 코드를 작성할 수 있다. 유지보수가 용이하며 확장성도 좋아진다. 또한 기능에 따라 코드를 분류하기 때문에 중복되는코드를 작성하는 실수 또한 줄일 수 있다.

# ORM 이란?
* ORM(Object Relational Mapping)
* 객체와 관계형 데이터베이스의 데이터를 자동으로 매핑(연결)해주는 것을 말한다.

**ORM을 사용하는 이유**
* 우리가 사용하는 객체지향 프로그래밍은 클래스를 사용하고, 관계형 데이터베이스는 테이블을 사용한다. 이 때 서로의 구조 간의 불일치가 발생하는데, ORM을 활용하여 객체 간의 관계를 바탕으로 SQL을 자동으로 생성하여 이러한 불일치를 해결할 수 있다.

**ORM의 장점**
* 익숙한 객체지향적인 코드를 사용함으로써 직관적인 이해 및 집중에 도움이 된다.
* ORM을 통해 작성한 객체를 재활용할 수 있다는 측면에서 재사용 및 유지보수의 편리성이 증가한다.

**ORM의 단점**
* 잘못 구현한 경우 성능 저하가 우려된다.
* 복잡한 쿼리문의 경우 오히려 SQL문으로 사용하는 것이 더 직관적이고 효율적일 수 있다.

## Sequelize, Sequelize CLI
* Node.js 기반의 ORM으로 PostgreSQL, MySQL, MariaDB, SQLite, MS-SQL을 지원한다.
* 프로미스 기반의 ORM이기 때문에 비동기 처리에 용이하다.

```javascript
SELECT * FROM Users WHERE id = 4  // SQL

let user = Users.find(4)  // Sequelize
```

**Sequelize CLI란?**
* 터미널에서 명령어를 사용해 데이터베이스 작업을 할 수 있는 툴이다.
* CLI의 대표적인 기능으로는 Migration, Seed, Model이 있다.
  1. Migration: 코드에 적혀 있는 데이터베이스 테이블에 대해 실제 데이터베이스에 테이블을 생성
  2. Seed: 생성된 테이블에 테이블을 추가
  3. Model: 테이블을 생성
