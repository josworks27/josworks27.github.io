# OOP(Obejct Oriented Programming)
OOP란?
> OOP의 기본 컨셉은 프로그램 내에서 표현하고자 하는 실 세계의 일들을 객체를 사용해서 모델링 하고, 객체를 사용하지 않으면 불가능 혹은 무지 어려웠던 일들을 쉽게 처리하는 방법을 제공하는 것이다.     -MDN-

생성자함수란?
* 생성자 함수는 함수가 가질 것 같은 모든 특징을 가지지만, 아무것도 리턴하지 않고 객체를 만들지도 않음
* 생성자 함수는 this를 사용하여 프로퍼티와 메서드를 정의함
* new 연사자를 붙여서 호출하면 생성자 함수로 동작함
* 생성자 함수의 첫 문자는 대문자로 하는 것이 관례임

```javascript
function Person(name) {
    this.name = name;
    this.greeting = () => console.log(`Hi I'm ${this.name}!`);
}

let person1 = new Person('JO');
```

* 매번 greeting() 함수를 다시 정의하면 메모리 효율적 측면에서 좋지 않음
* prototype에 함수를 정의하여 Person.prototype을 참조하는 모든 객체가 greeting() 메서드를 공유함

```javascript
function Person(name) {
    this.name = name;
}

Person.prototype.greeting = function () {
    console.log(`Hi I'm ${this.name}!`);
}

let person2 = new Person('NAKAO');

person2.greeting();
// Hi I'm NAKAO!
```

# Prototype
* JAvascript는 프로토타입 기반 언어(prototype-based language)라 불림
* 객체는 메소드와 속성들을 상송 받기 위한 템플릿으로써 프로토타입 객체(prototype object)를 가짐
* 프로토타입 객체도 또 다시 상위 프로토타입 객체로부터 메소드와 속성을 상속받을 수 있고 그 상위 프로토타입 객체도 마찬가지임
* 이를 프로토타입 체인(prototype chain)이라고 부름

## 1. 해당 함수에 Constructor(생성자) 자격 부여
* Constructor 자격을 부여 받아야만 new 연산자를 통해 객체를 만들 수 있음
* 이러한 이유로 함수만 new 키워드를 사용할 수 있음

## 2. 해당 함수의 Prototype Object 생성 및 연결
* 함수를 정의하면 함수만 생성되는 것이 아니라 Prototype Object도 같이 생성됨
* Prototype Object는 일반적인 객체와 같으며 디폴트 속성으로 constructor와 __proto__를 가지고 있음
* constructor는 생성자 함수 자체를 가리키며, __proto__는 객체가 생성될 떄 조상이었던 함수의 Prototype Object를 가리킴
