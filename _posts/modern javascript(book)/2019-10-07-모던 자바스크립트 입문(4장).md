# 4.1 객체의 기초 ① 객체 리터럴
자바스크립트에서는 원시 타입을 제외한 모든 값이 객체임

## 4.1.1 객체
* 객체는 이름과 값을 한 쌍으로 묶은 데이터를 여러 개 모은 것임. 즉, 데티터 여러 개를 하나로 모은 복합 데이터로 **연관 배열** 또는 **사전**이라고 부름
* 객체의 포함된 데이터 하나(이름과 쌍)을 객체의 **프로퍼티**라고 하며, 프로퍼티의 이름 부분을 **키**라고 함

## 4.1.2 객체 리터럴로 객체 생성하기
* {...} 부분이 객체 리터럴임

```javascript
var card = {
    suit: "하트",
    rank: "A"
};
```
* 객체 안의 프로퍼티 값을 읽거나 쓸 때는 마침표(.) 또는 대괄호([])를 사용함

```javascript
card.suit  // 하트
card["rank"]  // A  
```

* 객체 없는 프로퍼티를 읽으려고 하면 undefined를 반환함

## 4.1.3 프로퍼티 추가와 삭제
* 없는 프로퍼티 이름에 값을 대입하면 새로운 프로퍼티가 추가됨

```javascript
card.value = 14;
console.log(card);  // Object {suit: "하트", rank: "A", value: 14}
```

* delete 연산자를 사용하면 프로퍼티를 삭제할 수 있음

```javascript
delete card.rank;
console.log(card);  // Object {suit: "하트", value: 14}
```

## 4.1.4 in 연산자로 프로퍼티가 있는지 확인하기
* in 연산자를 사용하면 객체에 특정 프로퍼티가 있는지 확인할 수 있음. true/false로 반환함

```javascript
var card = { 
    suit: "하트",
    rank: "A"
}

console.log("suit" in card);  // true
console.log("color" in card);  // false
```

* 다만 in 연산자가 조사하는 대상이 해당 객체가 가진 프로퍼티와 그 객체가 **상속받은 모든 프로퍼티**라는 점을 주의해야 함

## 4.1.5 객체 리터럴 예제

```javascript
var p = { x: 1.0, y: 2.5 };

var circle = { 
    center: { x: 1.0, y: 2.0 },
    radius: 2.5 };

var person = {
    name: "조성철",
    age: 29,
    sex: "남",
    married: false
}
```

## 4.1.6 드서드
프로퍼티에 저장된 값의 타입이 함수면 그 프로퍼티를 **메서드**라고 함

## 4.1.7 객체는 참조 타입
객체 타입의 값을 변수에 대입하면 그 변수에는 객체의 참조(메모리에서의 위치 정보)가 저장됨

```javascript
var a = card;
```

* 변수 a가 card 객체를 참조하게 되므로 a로 card 객체를 읽거나 수정할 수 있음

# 4.2 함수의 기초

## 4.2.1 함수
* 자바스크립트에서 함수는 일련의 처리를 하나로 모아 언제든 호출할 수 있도록 만들어 둔 것임
* 함수의 입력 값을 **인수**, 출력 값을 **반환값**이라고 함

## 4.2.2 함수 선언문으로 함수 정의하기
function 키워드를 사용하여 정의함

```javascript
function square(x) {
    return x * x;
}
```

## 4.2.3 함수 이름
모든 식별자를 함수 이름으로 사용 가능하지만 함수의 기능을 이해하기 쉽게 지어야 함

## 4.2.4 함수 호출
함수를 호출하려면 함수 이름 뒤에 소괄호로 인수를 입력함

```javascript
square(3)  // 9
```

## 4.2.5 인수
* 함수는 인수를 여러 개 받을 수 있으며 인수가 여러 개라면 인수와 인수를 쉼표(,)로 구분함
* 인수가 없고 return 문도 없는 함수는 undefined가 반환됨

## 4.2.6 함수의 실행 흐름
1. 호출한 코드에 있는 인수가 정의문의 인자에 대입됨
2. 함수 정의문에 중괄호 안에 작성된 프로그램이 순차적으로 실행됨
3. return 문이 실행되면 호출한 코드로 돌아간다. return 문의 값은 함수의 반환값이 됨
4. return 문이 실행되지 않은 상태에로 마지막 문장이 실행되면, 호출한 코드로 돌아간 후에 undefined가 함수의 반환값이 됨

## 4.2.7 함수 선언문의 끌어올림
자바스크립트 엔진은 변수 선언문과 마찬가지로 함수 선언문을 프로그램의 첫머리로 끌어올림

## 4.2.8 값으로서의 함수
변수 값을 다른 변수에 할당하면 그 변수 이름으로 함수를 실행할 수 있음

```javascript
var sq = square;
console.log(sq(5))  // 25
```

## 4.2.9 참조에 의한 호출과 값에 의한 호출
함수는 원시 값을 인수로 넘겼을 때와 객체를 인수로 넘겼을 때 다르게 동작함

1. 인수에 원시 값을 넘기면 그 값 자체가 인자에 전달됨. 이를 **값의 전달**이라고 함
  * 변수 a와 변수 x는 다른 영역의 메모리에 위치한 별개의 변수이며 x값을 바꿔도 a값은 바뀌지 않음

```javascript
function add1(x) { return x = x + 1 };
var a = 3;
var b = add1(a);
console.log("a = " + a + ", b = " + b);  // a = 3, b = 4
```

2. 인수로 객체를 넘기면 전달되는 값은 참조 값임. 이를 **참조 전달**이라고 함
  * 인자 p와 변수 a는 똑같은 객체를 참조하고 있기 때문에 함수 안에서 p.x와 p.y를 수정하는 행위는 a.x와 a.y를 수정하는 것과 같음

```javascript
function add1(p) {
    p.x = p.x + 1;
    p.y = p.y + 1;
    return p;
}

var a = { x: 3, y: 4 };
var b = add1(a);
console.log(a, b);  // Object {x=4, y=5} Object {x=4, y=5}
```

## 4.2.10 변수의 유효 범위 ⭐️

### 전역 유효 범위와 지역 유효 범위
* 변수에 접근할 수 있는 범위를 그 변수의 유효 범위(Scope)라고 하며 자바스크립트는 어휘적 범위를 채택하고 있음
* 어휘적 범위(Lexical scope): 프로그램의 구문만으로 유효 범위를 정함
* 전역 변수의 유효 범위는 전체 프로그램이며, 지역 변수의 유효 범위는 함수 안에서 선언된 변수와 함수 인자로 변수가 선언된 함수 내부에 한정됨

### 변수의 충돌
* 변수에 유효 범위가 있는 이유는 프로그램의 다른 부분에서 선언된 이름이 같은 변수와 충돌하지 않도록 하기 위해서임
* 전역 변수와 지역 변수의 이름이 같아 충돌하는 경우, 전역 변수를 숨기고 지역 변수를 사용하게 됨

### 함수 안에서의 변수 선언과 변수 끌어올림
자바스크립트 엔진은 함수 안의 변수 선언부를 함수의 첫머리로 끌어올림

### 함수 안에서의 변수 선언 생략
변수를 선언하지 않은 상태에서 값을 대입하면 *전역 변수*로 선언됨

```javascript
function f() {
    a = "local";
    console.log(a);  // local
    return a;
}
```

## 4.2.11 블록 유효 범위: let과 const
* 블록 유효 범위를 가진 변수는 중괄호({}) 안에서만 유효하며, let은 변수를 선언하고 const는 한 번만 할당할 수 있는 상수를 선언함

### let 선언자
* var과 let으로 선언한 변수의 가장 큰 차이점은 let으로 선언한 변수의 유효 범위가 **블록 안**이라는 점임
* 자바스크립트 엔진은 var문과 달리 let문으로 선언한 변수를 끌어올림 하지 않음(호이스팅 X)

### const 선언자
* const문으로 선언한 상수는 let문으로 선언한 변수처럼 동작. 단, 반드시 초기화해야 함
* 상수 값이 객체이거나 배열일 경우에는 프로퍼티 또는 프로퍼티의 값을 수정 가능함

## 4.2.12 함수 리터럴로 함수 정의하기
* 함수 리터럴은 이름이 없는 함수이므로 **익명 함수** 또는 **무명 함수**라고 부름
* 함수 리터럴을 사용할 때는 긑에 반드시 세미콜론을 붙임
* 함수 러터럴로 정의한 함수는 끌어올리지 않음

```javascript
var square = function(x) { return x * x };
```

## 4.2.13 개체의 메서드
* 일반적으로 메서드가 속한 객체의 내부 데이터(프로퍼티 값)의 상태를 바꾸는 용도로 사용함
* 객체 지향 언어에서는 데이터와 그 상태를 바꾸는 메서드를 하나로 묶는 용도로 객체를 사용함

## 4.2.14 함수를 활용하면 얻을 수 있는 장점
1. 재사용할 수 있다.
2. 만든 프로그램을 이해하기 쉽다.
3. 프로그램 수정이 간단해진다.
